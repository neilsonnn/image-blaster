# Compendium

This project is an assortment of Claude skills for generating worlds with threejs. 

## Details

### Project Structure

1. Skills directory for Claude to use (https://code.claude.com/docs/en/skills) where we build and store the skills.
2. .env file for the API keys.
3. React app directory for the web application.
4. Working directory for Claude that the skills and React both use, not tracked by git.

### 1. Claude Skills

All skills use the following information:

#### Working directory structure

worlds/ - directory for the worlds generated by the skills.
worlds/world-name/ - directory for the world generated by the skill.
world/world-name/source/ - where the user can add source files for the skills to use.
worlds/world-name/world/ - directory for the create-world skill.
worlds/world-name/output/ - directory for any output generated by any other skills (image editing models, etc).

#### create-world

Uses the World Labs API to generate a world from which we can use the guassian splats (.spz), colliders (.glb) and panorama (.png). 
https://docs.worldlabs.ai/

This skill understands from the user request what files from the source directory are needed to generate the world.
It then uses the World Labs API to generate the world and creates the necessary files in the world directory to track the state of that world generation operation and then the final output from that operation.

### 2. React Project

The React project using Typescript, SparkJS (https://sparkjs.dev/), React Three Fibre (https://r3f.docs.pmnd.rs/), can load any world from the working directory and display it. 

#### Displaying worlds
The React project very elegantly can display any world we create inside the worlds/ directory. 
1. Each world is at minimum a guassian splat (.spz), colliders (.glb) and panorama (.png) which are displayed using Spark, an environment map (using the panorama), and as an invisible collider (using the colliders).
2. On the right side of the page is a column that lets us select any of the worlds we have created (we should have a thumbnail and a slug from the World Labs API). Worlds load in elegantly, when worlds load both on page load and when we switch, the splats, sound effects, and anything else gracefully fades out (if there was a world loaded), and then the new world loads in gracefully. Worlds fade using a reveal effect (https://sparkjs.dev/examples/#splat-reveal-effects). We have an efficient util function that handles all these. We will also have a basic character controller that when the scene is at 0,0,0 black we teleport back into the origin of the next world before we fade back in. 
3. If there are any audio files in the worlds/world-name/output/ directory, we play them looping in the background. 
4. We have a debug Leva panel (https://github.com/pmndrs/leva) we can use to tweak any parameters for post-processing, world loading speed, etc.
